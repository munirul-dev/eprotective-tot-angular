Open the browser console. Do you see the famous ExpressionChangedAfterItHasBeenCheckedError?

Scenario: We are trying to update the status property inside ngAfterViewInit.

Current Status: {{ status }}

Why? Angular verifies that bindings haven't changed after the view is composed. If they do, it throws this error to prevent unstable UI states.
Fix me: Use setTimeout or ChangeDetectorRef.detectChanges().
